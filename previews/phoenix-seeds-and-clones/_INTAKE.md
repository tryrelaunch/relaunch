### 3.2 What to protect (likely SEO-value pages, public-evidence basis)

- `/` — primary entity page, ranks for "phoenix marijuana clones" / "phoenix marijuana seeds" cluster
- `/the-vault/` — brand-search anchor; URL is referenced from a sitewide banner so internal-link equity is high
- `/germinate-marijuana-seeds/`, `/growing-marijuana-at-home/`, `/weed-seedlings-guide/`, `/sativa-vs-indica/`, `/keep-marijuana-seeds-fresh/` — the informational pages. - `/clones-new-mexico/` — has its own URL equity; even though it now points offsite, removing it would tank "new mexico clones" rankings

### 3.3 What to keep but improve

- `/` — keep architecture, but: refresh stale "2022" callouts in the Vault banner, fix the strain list (currently a wall of bold text — convert to a real table/section), add structured FAQ schema, fix the contradiction with `/clones-new-mexico/` re: consultation requirement.

- Each guide page (`/germinate-marijuana-seeds/`, etc.) — preserve current copy verbatim where it ranks, but enforce AI-Overview-ready answer-first restructure (TL;DR up top, then evidence).

### 3.4 Pages to create (gaps)

- `/strains/` — a single canonical landing for the rotating clone menu (currently the homepage embeds the strain list and the "rotated weekly" list as parallel content). Lets each individual strain page (if/when we build them) link to a parent.
- `/strains/[strain]/` — programmatic strain pages. Only build the top ~5–10 by demand. Each is a money page candidate (e.g. `/strains/blueberry-muffin/`). **Wait until clone demand data confirms which strains hold appointments before building all of them.**

- `/guides/` — index page for the existing how-to pages. Today they exist as orphans with no parent — internal linking can't flow.

### 3.9 Metadata plan

Each page needs unique title + meta description following the SEO playbook patterns. Critical fixes:

- **Title-tag year removal.** `/the-vault/` currently has "Hottest New Crosses For 2022" in the title — that's a ranking-killer for evergreen queries. Move year framing into a banner, not metadata.
- **Title-tag intent matching.** The homepage title is "Marijuana Clones Phoenix, AZ | Get High Yield Seeds & Clones." That's decent but the "high yield" modifier doesn't carry search volume. Test alternatives like "Phoenix Marijuana Clones & Seeds | $30 Clones, In-Person Appointments."
- **Meta-description CTAs.** Several meta descriptions are missing CTA language. Every commercial page should end with a verb ("Book your appointment," "Order Vault seeds today").
- **Robots directives.** All current pages are `index, follow` — good. After the rebuild, keep that for all customer-facing pages; `noindex` only `/cart/`, `/checkout/`, `/my-account/`, `/order-tracking/`, and the Calendly thank-you page.

### 3.10 Schema plan

Cannabis schema is sensitive — over-claiming licensure or product details can trigger Google manual actions. Conservative implementation:

- **Homepage:** `LocalBusiness` (NOT `Store` or `Dispensary`) + `FAQPage` + `WebSite`.
- **`/locations/tempe/`:** `LocalBusiness` with address, hours, geo coordinates.
- **`/the-vault/`, `/grow-tent-kit-phoenix/`:** `CollectionPage` parent + individual `Product` schema on each product page (price, availability, no THC-content claims in schema).
- **All `/guides/*`:** `Article` + `BreadcrumbList`.
- **`/faq/`:** `FAQPage`.
- **`/about/`:** `AboutPage` + `Person` for each named team member.
- **Sitewide:** `BreadcrumbList` everywhere except homepage.

**Do NOT add:** any schema that asserts THC content, "medical" claims, or licensure not verified.
