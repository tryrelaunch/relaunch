## 4. Schema Markup Rules (Corrected)

### Required schema per page type
| Page type | Schema |
|---|---|
| Homepage | `LocalBusiness` (or appropriate subtype) + `Organization` + `BreadcrumbList` |
| Service page | `Service` + `BreadcrumbList` |
| FAQ-bearing page | `FAQPage` (only if real Q&A is on the page) |
| Pricing page | `Service` + `Offer` (only with real, visible prices) |
| Contact page | `ContactPage` |

### What NOT to ship
- **`AggregateRating`** — only ship if real reviews are *visibly displayed on the same page*. Google removed star snippets for self-serving review schema. Don't include AggregateRating just because the brand has reviews on Yelp/Google. Either embed 3–5 quoted reviews on the page AND mark them up, or drop the AggregateRating block.
- **`Product` / `Offer` with placeholder prices** — never. If pricing is quote-based, omit Offer schema entirely.
- **`Event` schema for generic charter availability** — only for real scheduled events (NYE, 4th of July, etc.)
- **`FAQPage` rich-snippet expectation** — Google removed FAQ rich results for non-authority sites in 2023. Include FAQPage schema for general structured-data + LLM-citation value, but don't expect SERP real estate from it.

## 5. Internal Linking Rules

1. **Within Malarky:** every commercial subpage links back to homepage in body copy, plus 2–3 sibling pages with descriptive anchor text (never exact-match keyword stuffing).
2. **Cross-fleet:** allowed only when it helps the user, contextually, in body copy or FAQ. Examples:
   - Malarky FAQ Q1 (group too big): "If your group is larger than 12, our sister vessel [Triton Charters](https://triton-charters.com) accommodates up to 100 guests."
   - Triton FAQ (group too small / wants intimate): "For groups of 12 or fewer who'd prefer a private intimate experience, see [Malarky Charters](https://malarkycharters.com)."
3. **Forbidden:** sitewide footer links across all 5 sister sites. Do not build a `sandiegoboatingguide.com` hub or any other PBN-style link aggregator. Editorial-context links only, no sitewide footers, no exact-match anchor stuffing.
4. **Anchor text:** brand-name or natural-language. Never "best yacht rental san diego" or other exact-match keyword stuff.
5. **Maximum 1–2 cross-site links per page.** More than that triggers Google's link-spam pattern recognition for related-business networks.

## 6. Content Quality Rules

1. **Write enough to satisfy search intent.** In the SD charter niche, top-ranking competitor pages run 1,200–2,500 words. Pages under 800 words won't rank. Pages over 3,000 words risk diluting intent. Calibrate per page; don't pad.
2. **Original photos always.** Generic stock-style images hurt trust signals. Use the boat library (Malarky has 26 boat photos already imported into `/images/boat/`).
3. **Concrete details over fluff.** "47-foot Voyage performance catamaran built in Cape Town, South Africa" beats "luxury yacht." Include real specs, real route, real add-on prices when available.
4. **No "best" / "#1" / "top-rated" claims unless provable** with a third-party citation (e.g., "5.0 stars on Google with 38 verified reviews on Yelp" is fine; "the best yacht in San Diego" is not).
5. **No "rank in X days" or "zero competition" claims** — about anything, ever. Search rankings depend on backlinks, freshness, indexing, and Google's whim. Don't promise timelines in copy or in cross-references.
6. **Every page must answer real booking questions:** price (or quote process), capacity, route, duration, pickup location, what's included, BYOB rules, captain details, cancellation policy, parking, what to bring.
7. **Two CTAs per page minimum:** above the fold and near the bottom. CTA target = FareHarbor booking flow. Phone number prominent.
8. **Catering FAQ on every commercial page** — Malarky offers catering through partners but doesn't advertise it. Add as FAQ Q&A, not in hero copy.

## 7. Local SEO Rules

1. **NAP consistency required across all 5 boat sites + GBP + Yelp + TripAdvisor + Viator + GetMyBoat + sandiego.org + every directory.**
2. **Google Business Profile must be fully optimized:** primary category "Boat Tour Agency" or "Boat Rental Service"; secondary "Tour Operator"; description uses all 750 chars; minimum 20 photos; services listed; Q&A pre-populated; weekly posts; booking link to FareHarbor.
3. **Review velocity:** post-charter automated review request via FareHarbor (email at +24hr, SMS at +72hr if no review). Crew mentions reviews verbally at end of charter. Respond to every Google review.
4. **Never gate reviews** (asking only happy customers) — violates Google policy. Send to all customers equally.
5. **Citations to build for Malarky specifically:** Boatsetter, GetMyBoat, TripAdvisor (already listed — optimize), Viator, sandiego.org tourism authority, Apple Maps, Bing Places, WeddingWire (for micro-wedding/elopement page).

## 8. Technical SEO Rules

1. **Title tag:** ≤60 chars, unique per page, includes primary keyword + brand
2. **Meta description:** 140–155 chars, unique per page
3. **One H1 per page** with primary keyword
4. **H2/H3 hierarchy** logical and keyword-themed
5. **Canonical URL** on every page
6. **WebP images** wherever practical, lazy-load below the fold
7. **Mobile-first** layout — Malarky's audience books on mobile
8. **Sitemap.xml + robots.txt at the root** (currently missing for `clients/malarky/` build — add)
9. **Booking widget loads non-blocking** so main page renders first
